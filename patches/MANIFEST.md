## Applying the patches to a fresh vendor drop

```sh
# 1. Get the vendor tarball
curl -O http://heirloom.sourceforge.net/... /heirloom-pkgtools-070227.tar.bz2
tar xjf heirloom-pkgtools-070227.tar.bz2
cd heirloom-pkgtools-070227

# 2. Initialise git for patch application
git init && git add -A && git commit -m "vendor: pristine heirloom-pkgtools-070227"

# 3. Apply the patches in order
git am /path/to/heirloom-pkgtools-darwin/patches/*.patch
```

Or as a single diff (loses per-commit metadata but works for
systems without `git am`):

```sh
cd heirloom-pkgtools-070227
patch -p1 < /path/to/heirloom-pkgtools-darwin/patches/cumulative.diff
```

## Regeneration

Automated. Regenerated by `sh /tmp/heirloom-patches.sh pkgtools ...`
each time the port receives new patches. Do not hand-edit patches/;
edit source files and let the script re-emit.
